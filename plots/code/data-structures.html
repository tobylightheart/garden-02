Data Structures

The Building Blocks of Efficient Code

Data structures are the foundation of efficient software. Choosing the right data structure for a problem can make the difference between code that runs in seconds versus hours. Understanding these structures is essential for any programmer.

Fundamental Operations

When analyzing data structures, we care about four key operations:

  • Create: How much to allocate?
  • Read: How fast to access?
  • Update: How fast to modify?
  • Delete: How fast to remove?
The best structure depends on which operations you need most.

Arrays

Fixed-size contiguous memory blocks.

Time complexity:

  • Access: O(1) - Direct calculation
  • Insertion: O(n) - Shift elements
  • Deletion: O(n) - Shift elements

Best for: Fast access, numerical data, dense storage

Linked Lists

Nodes connected by pointers/references.

Time complexity:

  • Access: O(n) - Must traverse
  • Insertion: O(1) - If you have the pointer
  • Deletion: O(1) - If you have the pointer

Best for: Frequent insertions/deletions, sparse data

Stacks

LIFO (Last In, First Out) - like a stack of plates.

Operations: push (add), pop (remove), peek (view top)

Time complexity: All operations O(1)

Use cases: Function call recursion, undo operations, expression parsing

Queues

FIFO (First In, First Out) - like a line at a store.

Operations: enqueue (add to back), dequeue (remove from front)

Time complexity: O(1) for both operations

Use cases: Task scheduling, breadth-first search, buffering

Hash Tables

Key-value stores using hash functions.

Time complexity:

  • Access: O(1) average (with good hash function)
  • Insertion: O(1) average
  • Deletion: O(1) average

Best for: Fast lookups, caching, dictionaries

Challenge: Collisions require handling

Trees

Hierarchical structures with parent-child relationships.

  • Binary Trees: Each node has max 2 children
  • BST (Binary Search Trees): Left < node < Right
  • Avg. search: O(log n)

Use cases: File systems, expression trees, databases

Heaps

Specialized complete binary trees.

  • Max-Heap: Parent >= children (priority queue)
  • Min-Heap: Parent <= children

Operations: insert O(log n), extract-min/max O(log n)

Use cases: Priority queues, Dijkstra's algorithm, heapsort

Graphs

Nodes (vertices) connected by edges.

  • Undirected: Edges have no direction
  • Directed: Edges have direction (A → B)
  • Weighted: Edges have associated values

Use cases: Networks, maps, social graphs, dependency resolution

Sets and Maps

Sets: Unique elements, no duplicates

Maps: Key-value associations (hash tables under the hood)

Time complexity: O(1) for operations on average

🤔 Choosing the Right Structure

Ask yourself:

  1. What operations matter most? Access? Insertion? Both?
  2. How big is your data? Small enough for O(n)?
  3. Do you need ordered data? Sorted access?
  4. Memory constraints? Pointers vs. contiguous arrays?
  5. Access patterns? Sequential or random?

Remember: There's no "best" structure, only the best fit for your problem.

Custom Linked List
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None
    
    def append(self, data):
        """Add to end: O(1) with tail reference"""
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        
        current = self.head
        while current.next:
            current = current.next
        current.next = new_node
    
    def prepend(self, data):
        """Add to front: O(1)"""
        new_node = Node(data)
        new_node.next = self.head
        self.head = new_node
    
    def search(self, data):
        """Search for data: O(n)"""
        current = self.head
        while current:
            if current.data == data:
                return True
            current = current.next
        return False

# Test
ll = LinkedList()
ll.append(10)
ll.append(20)
ll.prepend(5)
print(f"Search 20: {ll.search(20)}")
print(f"Search 100: {ll.search(100)}")
Binary Search Tree
class BSTNode:
    def __init__(self, data):
        self.data = data
        self.left = None
        self.right = None

class BST:
    def __init__(self):
        self.root = None
    
    def insert(self, data):
        """Insert maintaining BST property: O(log n) avg"""
        if not self.root:
            self.root = BSTNode(data)
            return
        
        current = self.root
        while True:
            if data < current.data:
                if not current.left:
                    current.left = BSTNode(data)
                    return
                current = current.left
            elif data > current.data:
                if not current.right:
                    current.right = BSTNode(data)
                    return
                current = current.right
            else:
                # Duplicate
                return
    
    def search(self, data):
        """Search in BST: O(log n) avg"""
        current = self.root
        while current:
            if data == current.data:
                return True
            elif data < current.data:
                current = current.left
            else:
                current = current.right
        return False
    
    def inorder(self):
        """In-order traversal: O(n)"""
        result = []
        self._inorder_recursive(self.root, result)
        return result
    
    def _inorder_recursive(self, node, result):
        if node:
            self._inorder_recursive(node.left, result)
            result.append(node.data)
            self._inorder_recursive(node.right, result)

# Test
bst = BST()
for num in [50, 30, 70, 20, 40, 60, 80]:
    bst.insert(num)
print(f"In-order (sorted): {bst.inorder()}")
print(f"Search 40: {bst.search(40)}")
print(f"Search 25: {bst.search(25)}")
Priority Queue (Heap)
import heapq

class PriorityQueue:
    def __init__(self):
        self.heap = []
    
    def push(self, item, priority):
        """Add item with priority: O(log n)"""
        # Lower priority number = higher priority
        heapq.heappush(self.heap, (priority, item))
    
    def pop(self):
        """Remove highest priority item: O(log n)"""
        if not self.heap:
            raise IndexError("Priority queue is empty")
        return heapq.heappop(self.heap)[1]
    
    def peek(self):
        """View highest priority without removing: O(1)"""
        if not self.heap:
            raise IndexError("Priority queue is empty")
        return self.heap[0][1]

# Test
pq = PriorityQueue()
pq.push("task3", 3)
pq.push("task1", 1)
pq.push("task2", 2)
print(f"Next task: {pq.pop()}")  # task1 (highest priority)
print(f"Next task: {pq.pop()}")  # task2
print(f"Next task: {pq.pop()}")  # task3
Graph Representation
from collections import defaultdict

class Graph:
    def __init__(self):
        self.adjacency_list = defaultdict(list)
    
    def add_edge(self, u, v):
        """Add directed edge: O(1)"""
        self.adjacency_list[u].append(v)
    
    def neighbors(self, node):
        """Get all neighbors: O(degree of node)"""
        return self.adjacency_list[node]
    
    def bfs(self, start):
        """Breadth-first search: O(V + E)"""
        visited = set()
        queue = [start]
        visited.add(start)
        
        while queue:
            node = queue.pop(0)
            for neighbor in self.adjacency_list[node]:
                if neighbor not in visited:
                    visited.add(neighbor)
                    queue.append(neighbor)
        
        return visited

# Test
g = Graph()
g.add_edge(0, 1)
g.add_edge(0, 2)
g.add_edge(1, 3)
g.add_edge(2, 3)
reachable = g.bfs(0)
print(f"Nodes reachable from 0: {reachable}")
Hash Table with Collision Handling
class HashTable:
    def __init__(self, capacity=16):
        self.capacity = capacity
        self.table = [[] for _ in range(capacity)]
    
    def _hash(self, key):
        """Simple hash function"""
        return hash(key) % self.capacity
    
    def insert(self, key, value):
        """Insert or update: O(1) average"""
        index = self._hash(key)
        bucket = self.table[index]
        
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)  # Update
                return
        
        bucket.append((key, value))  # Insert
    
    def get(self, key):
        """Get value: O(1) average"""
        index = self._hash(key)
        bucket = self.table[index]
        
        for k, v in bucket:
            if k == key:
                return v
        return None
    
    def delete(self, key):
        """Delete key: O(1) average"""
        index = self._hash(key)
        bucket = self.table[index]
        
        for i, (k, v) in enumerate(bucket):
            if k == key:
                del bucket[i]
                return True
        return False

# Test
ht = HashTable()
ht.insert("apple", 1)
ht.insert("banana", 2)
ht.insert("orange", 3)
print(f"Get banana: {ht.get('banana')}")
ht.delete("banana")
print(f"Get banana after delete: {ht.get('banana')}")
