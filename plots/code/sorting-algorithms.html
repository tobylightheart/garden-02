Sorting Algorithms

Ordering Data Efficiently

Sorting is one of the most fundamental operations in computer science. From organizing a phone book to ranking search results, sorting appears everywhere. Understanding different sorting algorithms helps you choose the right tool for each problem.

Big O Notation Basics

When analyzing sorting algorithms, we care about time complexity:

  • O(n): Linear - fast for large data
  • O(n log n): Linearithmic - very efficient
  • O(n²): Quadratic - slow for large data
  • O(2ⁿ): Exponential - impractical for large data

Bubble Sort

Simple but inefficient. Repeatedly swap adjacent elements if they're in wrong order.

Time complexity: O(n²) average and worst case

Best for: Educational purposes, tiny datasets

Why it's slow: Too many comparisons and swaps

Selection Sort

Find minimum, swap to front. Repeat n-1 times.

Time complexity: O(n²) always

Best for: Very small arrays

Why it's slow: Doesn't learn from previous passes

Insertion Sort

Build sorted array one element at a time. Like sorting cards in hand.

Time complexity: O(n²) worst, O(n) best (already sorted)

Best for: Small arrays, nearly sorted data

Why it's good: Simple, stable, adaptive

Merge Sort

Divide and conquer. Split, sort halves, merge.

Time complexity: O(n log n) always

Best for: Large datasets, external sorting (disk)

Pros: Consistent performance, stable

Cons: Uses O(n) extra space

Quick Sort

Divide and conquer. Pick pivot, partition, sort sides.

Time complexity: O(n log n) average, O(n²) worst

Best for: General-purpose sorting

Pros: Fast in practice, in-place

Cons: Worst case on sorted data

Trick: Randomize pivot to avoid worst case

Heap Sort

Uses heap data structure. Build heap, repeatedly extract max.

Time complexity: O(n log n) always

Best for: When guaranteed performance matters

Pros: Consistent O(n log n), in-place

Cons: Not stable, complex to implement

Radix Sort

Sort by digits/characters. Bucket by digit, sort buckets.

Time complexity: O(nk) where k is number of digits

Best for: Integers, strings of fixed length

Pros: Linear time, stable

Cons: Only works on specific data types

Counting Sort

Count frequency of each value. Reconstruct sorted array.

Time complexity: O(n + k) where k is range

Best for: Small integer ranges

Pros: Linear time, stable

Cons: Only for integers with limited range

🤔 Choosing a Sort

For production code: Use built-in sort (Timsort in Python, Quicksort in C++, Introsort in Java). They're optimized and well-tested.

When to implement custom sorts:

  • Need specific stability requirements
  • Working with special data types
  • Learning and understanding
  • Parallel sorting

QuickSort vs MergeSort:

  • QuickSort: Faster in practice, in-place
  • MergeSort: Stable, predictable performance

💻 Python Implementations

🔸 Bubble Sort

def bubble_sort(arr):
    """Bubble sort: repeatedly swap adjacent elements"""
    n = len(arr)
    arr = arr.copy()  # Don't modify original
    
    for i in range(n):
        swapped = False
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                # Swap
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:
            break  # Already sorted!
    
    return arr

# Test
data = [64, 34, 25, 12, 22, 11, 90]
print(f"Original: {data}")
print(f"Sorted:   {bubble_sort(data)}")
print(f"Comparisons: O(n²) worst case")

🔸 Selection Sort

def selection_sort(arr):
    """Selection sort: find minimum, swap to front"""
    n = len(arr)
    arr = arr.copy()
    
    for i in range(n):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        # Swap
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    
    return arr

# Test
data = [64, 34, 25, 12, 22, 11, 90]
print(f"Original: {data}")
print(f"Sorted:   {selection_sort(data)}")
print(f"Always O(n²) comparisons and swaps")

🔸 Insertion Sort

def insertion_sort(arr):
    """Insertion sort: build sorted array one element at a time"""
    n = len(arr)
    arr = arr.copy()
    
    for i in range(1, n):
        key = arr[i]
        j = i - 1
        
        # Move elements greater than key
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    
    return arr

# Test - Already sorted (best case)
data_sorted = [11, 12, 22, 25, 34, 64, 90]
print(f"Already sorted: {insertion_sort(data_sorted)}")

# Test - Reverse sorted
data_reverse = [90, 64, 34, 25, 22, 12, 11]
print(f"Reverse sorted: {insertion_sort(data_reverse)}")
print(f"Best case: O(n) when already sorted")

🔸 Merge Sort

def merge_sort(arr):
    """Merge sort: divide and conquer with guaranteed O(n log n)"""
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)

def merge(left, right):
    """Merge two sorted arrays"""
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    # Append remaining elements
    result.extend(left[i:])
    result.extend(right[j:])
    
    return result

# Test
data = [64, 34, 25, 12, 22, 11, 90, 45]
print(f"Original: {data}")
print(f"Sorted:   {merge_sort(data)}")
print(f"Consistent O(n log n) - stable sort")

🔸 Quick Sort

def quick_sort(arr, low=0, high=None):
    """Quick sort: divide and conquer with randomized pivot"""
    arr = arr.copy()
    n = len(arr)
    
    if high is None:
        high = n - 1
    
    if low < high:
        # Random pivot to avoid worst case
        import random
        pivot_idx = random.randint(low, high)
        pivot_idx = partition(arr, low, high, pivot_idx)
        
        quick_sort(arr, low, pivot_idx - 1)
        quick_sort(arr, pivot_idx + 1, high)
    
    return arr

def partition(arr, low, high, pivot_idx):
    """Lomuto partition scheme"""
    # Move pivot to end
    pivot = arr[pivot_idx]
    arr[pivot_idx], arr[high] = arr[high], arr[pivot_idx]
    
    store_idx = low
    for i in range(low, high):
        if arr[i] < pivot:
            arr[store_idx], arr[i] = arr[i], arr[store_idx]
            store_idx += 1
    
    # Move pivot to correct position
    arr[store_idx], arr[high] = arr[high], arr[store_idx]
    return store_idx

# Test
data = [64, 34, 25, 12, 22, 11, 90, 45]
print(f"Original: {data}")
print(f"Sorted:   {quick_sort(data)}")
print(f"Average O(n log n), worst O(n²)")

🔸 Heap Sort

def heap_sort(arr):
    """Heap sort: build max-heap, extract max repeatedly"""
    arr = arr.copy()
    n = len(arr)
    
    # Build max heap
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)
    
    # Extract elements
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]
        heapify(arr, i, 0)
    
    return arr

def heapify(arr, n, i):
    """Ensure max-heap property at index i"""
    largest = i
    left = 2 * i + 1
    right = 2 * i + 2
    
    if left < n and arr[left] > arr[largest]:
        largest = left
    
    if right < n and arr[right] > arr[largest]:
        largest = right
    
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)

# Test
data = [64, 34, 25, 12, 22, 11, 90, 45]
print(f"Original: {data}")
print(f"Sorted:   {heap_sort(data)}")
print(f"Consistent O(n log n) - not stable")

🔸 Radix Sort

def counting_sort_by_digit(arr, exp):
    """Sort array by digit at position exp (1, 10, 100, ...)"""
    n = len(arr)
    output = [0] * n
    count = [0] * 10
    
    # Count frequencies
    for num in arr:
        index = (num // exp) % 10
        count[index] += 1
    
    # Cumulative count
    for i in range(1, 10):
        count[i] += count[i - 1]
    
    # Build output (iterate backwards for stability)
    for num in reversed(arr):
        index = (num // exp) % 10
        output[count[index] - 1] = num
        count[index] -= 1
    
    return output

def radix_sort(arr):
    """Radix sort: sort by each digit from least to most significant"""
    arr = arr.copy()
    
    # Find maximum number
    max_val = max(arr)
    
    # Sort by each digit
    exp = 1
    while max_val // exp > 0:
        arr = counting_sort_by_digit(arr, exp)
        exp *= 10
    
    return arr

# Test - Positive integers
data = [170, 45, 75, 90, 802, 24, 2, 66]
print(f"Original: {data}")
print(f"Sorted:   {radix_sort(data)}")
print(f"O(nk) time - stable, only for integers")

🔸 Counting Sort

def counting_sort(arr):
    """Counting sort: count frequency, reconstruct"""
    arr = arr.copy()
    
    if len(arr) == 0:
        return arr
    
    # Find range
    min_val = min(arr)
    max_val = max(arr)
    range_size = max_val - min_val + 1
    
    # Count frequencies
    count = [0] * range_size
    for num in arr:
        count[num - min_val] += 1
    
    # Reconstruct sorted array
    sorted_arr = []
    for i in range(range_size):
        sorted_arr.extend([i + min_val] * count[i])
    
    return sorted_arr

# Test - Small integer range
data = [4, 2, 2, 8, 3, 3, 1]
print(f"Original: {data}")
print(f"Sorted:   {counting_sort(data)}")
print(f"O(n + k) - only for limited integer ranges")
