Sorting Algorithms

Ordering Data Efficiently

Sorting is one of the most fundamental operations in computer science. From organizing a phone book to ranking search results, sorting appears everywhere. Understanding different sorting algorithms helps you choose the right tool for each problem.

Big O Notation Basics

When analyzing sorting algorithms, we care about time complexity:

  • O(n): Linear - fast for large data
  • O(n log n): Linearithmic - very efficient
  • O(n²): Quadratic - slow for large data
  • O(2ⁿ): Exponential - impractical for large data

Bubble Sort

Simple but inefficient. Repeatedly swap adjacent elements if they're in wrong order.

Time complexity: O(n²) average and worst case

Best for: Educational purposes, tiny datasets

Why it's slow: Too many comparisons and swaps

Selection Sort

Find minimum, swap to front. Repeat n-1 times.

Time complexity: O(n²) always

Best for: Very small arrays

Why it's slow: Doesn't learn from previous passes

Insertion Sort

Build sorted array one element at a time. Like sorting cards in hand.

Time complexity: O(n²) worst, O(n) best (already sorted)

Best for: Small arrays, nearly sorted data

Why it's good: Simple, stable, adaptive

Merge Sort

Divide and conquer. Split, sort halves, merge.

Time complexity: O(n log n) always

Best for: Large datasets, external sorting (disk)

Pros: Consistent performance, stable

Cons: Uses O(n) extra space

Quick Sort

Divide and conquer. Pick pivot, partition, sort sides.

Time complexity: O(n log n) average, O(n²) worst

Best for: General-purpose sorting

Pros: Fast in practice, in-place

Cons: Worst case on sorted data

Trick: Randomize pivot to avoid worst case

Heap Sort

Uses heap data structure. Build heap, repeatedly extract max.

Time complexity: O(n log n) always

Best for: When guaranteed performance matters

Pros: Consistent O(n log n), in-place

Cons: Not stable, complex to implement

Radix Sort

Sort by digits/characters. Bucket by digit, sort buckets.

Time complexity: O(nk) where k is number of digits

Best for: Integers, strings of fixed length

Pros: Linear time, stable

Cons: Only works on specific data types

Counting Sort

Count frequency of each value. Reconstruct sorted array.

Time complexity: O(n + k) where k is range

Best for: Small integer ranges

Pros: Linear time, stable

Cons: Only for integers with limited range

🤔 Choosing a Sort

For production code: Use built-in sort (Timsort in Python, Quicksort in C++, Introsort in Java). They're optimized and well-tested.

When to implement custom sorts:

  • Need specific stability requirements
  • Working with special data types
  • Learning and understanding
  • Parallel sorting

QuickSort vs MergeSort:

  • QuickSort: Faster in practice, in-place
  • MergeSort: Stable, predictable performance
