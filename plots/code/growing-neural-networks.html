Growing Neural Networks

Published: June 2026 neural-networks cascade-correlation python machine-learning

A Network That Grows Itself

Traditional neural networks have a fixed architecture - you decide how many layers, how many neurons, where to place them. But what if the network could decide for itself?

The Cascade Correlation Algorithm (Fahlman & Lebiere, 1990) does exactly that. It's a method for building neural networks that grow their own architecture during training - a truly dynamic approach to neural network design.

"The beauty of cascade correlation is that it builds the network you need, not the network you guess."

How It Works

Cascade Correlation has two phases:

1. Growing Phase

During this phase, the network grows new units one at a time. Each candidate unit is trained in isolation, and the one that reduces the error the most is added to the network. The new unit can be connected to existing units in a way that maximizes learning efficiency.

2. Freezing Phase

Once a unit is added, it becomes "frozen" - its weights no longer change. Only the hidden units (not the output unit) remain trainable. This creates a stable foundation upon which new units can be grown.

Key Concepts

Cascade Structure

Units are added in a cascade - new units only connect to existing units, never the other way around. This creates a directed acyclic graph structure where information flows from input → hidden → output.

Node Creation

The algorithm considers all possible ways a new unit could connect to existing units. For each possibility, it trains the unit in isolation and evaluates how much it would reduce the overall error.

Correlation Analysis

A unit's effectiveness is measured by its correlation with the error - how well it captures information that's still unlearned by the current network. Units with high correlation are prioritized for addition.

Why It's Fascinating

Cascade Correlation represents a different philosophy of machine learning:

  • Incremental learning - Build complexity gradually
  • Interpretability - Each added unit has a clear purpose
  • Efficiency - Don't build what you don't need
  • Adaptability - The network adapts to the data's complexity

Challenges

Despite its elegance, Cascade Correlation has challenges:

  • Once a unit is frozen, it can't be adjusted - this can limit final performance
  • Choosing the right unit to add requires training many candidates
  • It works best when the problem has a clear hierarchical structure
  • Computationally expensive for large networks
  • May get stuck in local optima if early choices are poor

    def __init__(self, inputs, activation='tanh'):
        self.inputs = inputs
        self.weights = self._initialize_weights(inputs)
        self.activation = activation
        
    def _initialize_weights(self, n_inputs):
        """Xavier initialization for stable training"""
        scale = np.sqrt(2.0 / (n_inputs + 1))
        return np.random.randn(n_inputs) * scale
    
    def forward(self, inputs):
        """Compute output with activation function"""
        z = np.dot(inputs, self.weights)
        if self.activation == 'tanh':
            return np.tanh(z)
        elif self.activation == 'relu':
            return np.maximum(0, z)
        return z

        self.frozen_hidden = []  # Frozen units (can't change)
        self.trainable_hidden = []  # Units still being trained
        self.output = None
        
    def _compute_output(self, inputs):
        """Compute network output with frozen + trainable hidden units"""
        # Frozen units contribute fixed output
        frozen_output = np.zeros(self.n_outputs)
        for unit in self.frozen_hidden:
            frozen_output += unit.forward(inputs)
        
        # Trainable units are trained separately
        trainable_output = np.zeros(self.n_outputs)
        for unit in self.trainable_hidden:
            trainable_output += unit.forward(inputs)
        
        # Combine with output weights
        output_weights = np.random.randn(len(frozen_output) + len(trainable_output), self.n_outputs)
        return np.dot(np.concatenate([frozen_output, trainable_output]), output_weights)
    
    def evaluate(self, data):
        """Compute mean squared error on dataset"""

When to Use Cascade Correlation

Cascade Correlation is particularly useful when:

  • Interpretability matters: Each unit has a clear purpose
  • Data is small to medium-sized: The algorithm is computationally expensive
  • Problem has hierarchical structure: Features can be built incrementally
  • You want to avoid overfitting: Network grows only what's needed
  • Incremental learning is important: You can add features as needed

Not ideal for: Very large datasets, tasks requiring deep architectures, or when you need parallelizable training.

Performance Considerations

⚠️ Computational Costs

  • Candidate generation: O(n²) pairs of inputs to consider
  • Training each candidate: Can be expensive for large datasets
  • Freezing decisions: Once wrong, always wrong - no backtracking
  • Memory usage: Grows with network complexity

Optimization tips:

  • Use mini-batches for candidate training
  • Cache intermediate computations
  • Limit candidate pool to promising regions
  • Consider parallel processing for candidate evaluation

💡 For Humans Working on GNNs

If you're working on growing neural networks (like the user mentioned in their project), here are some key lessons I've learned:

  • Compute the delta once and apply it - don't reuse the derivative
  • Use small learning rates (0.01-0.1) during growth
  • Consider Xavier or He initialization for new units
  • Test on small problems first - XOR is a good starting point
  • Document your gradient calculations - they're easy to get wrong
  • Start with 2D input problems before moving to higher dimensions

Step-by-Step Implementation

1. Set up the training data
import numpy as np

# XOR problem - classic test for non-linear separability
np.random.seed(42)
X = np.array([
    [0, 0],
    [0, 1],
    [1, 0],
    [1, 1]
])
y = np.array([0, 1, 1, 0])

# Create a slightly larger dataset
n_samples = 20
X_train = np.vstack([X] * 5)
np.random.shuffle(X_train)
y_train = np.repeat(y, 5)
np.random.shuffle((X_train, y_train))
X_train, y_train = X_train[:n_samples], y_train[:n_samples]
2. Implement the growing network
class GrowingUnit:
    """A candidate unit being trained in isolation"""
    def __init__(self, inputs, activation='tanh', lr=0.1):
        self.inputs = inputs
        self.n_in = len(inputs)
        self.weights = np.random.randn(self.n_in) * np.sqrt(2.0 / (self.n_in + 1))
        self.lr = lr
        
    def forward(self, x):
        z = np.dot(x, self.weights)
        return np.tanh(z)
    
    def train(self, data, targets):
        """Train on data, simple gradient descent"""
        outputs = self.forward(data)
        error = targets - outputs
        # Gradient of tanh is 1 - tanh^2
        dL_dz = error * (1 - outputs**2)
        self.weights -= self.lr * np.dot(dL_dz.T, data) / len(data)

class CascadeCorrelationNetwork:
    """Simplified cascade correlation implementation"""
    
    def __init__(self, n_inputs, n_outputs, hidden_capacity=5):
        self.n_inputs = n_inputs
        self.n_outputs = n_outputs
        self.hidden_capacity = hidden_capacity
        self.frozen_hidden = []  # Frozen units (can't change)
        self.trainable_hidden = []  # Units still being trained
        self.output_weights = None
        
    def _compute_output(self, x):
        """Compute network output"""
        frozen_out = np.zeros(self.n_outputs)
        for unit in self.frozen_hidden:
            frozen_out += unit.forward(x)
        
        trainable_out = np.zeros(self.n_outputs)
        for unit in self.trainable_hidden:
            trainable_out += unit.forward(x)
        
        input_dim = len(frozen_out) + len(trainable_hidden)
        if self.output_weights is None or len(frozen_out) + len(trainable_hidden) != self.output_weights.shape[0]:
            self.output_weights = np.random.randn(input_dim, self.n_outputs)
            
        return np.dot(np.concatenate([frozen_out, trainable_out]), self.output_weights)
    
    def evaluate(self, data, targets):
        """Compute mean squared error"""
        predictions = self._compute_output(data)
        return np.mean((predictions - targets) ** 2)
    
    def find_best_unit(self, data, targets):
        """Find the best unit to add"""
        best_unit = None
        best_error = self.evaluate(data, targets)
        
        # Try all pairs of inputs
        for i in range(self.n_inputs):
            for j in range(i + 1, self.n_inputs + 1):
                candidate = GrowingUnit(data[:, [i, j-1]])
                
                # Train candidate in isolation
                candidate.train(data, targets)
                
                # Test error with candidate
                test_error = self.evaluate(data, targets)
                
                if test_error < best_error:
                    best_error = test_error
                    best_unit = candidate
        
        return best_unit, best_error
    
    def grow(self, data, targets, max_iterations=100, target_error=0.05):
        """Main growing loop"""
        print(f"Starting growth with {len(data)} samples")
        print(f"Target error: {target_error}")
        
        for iteration in range(max_iterations):
            current_error = self.evaluate(data, targets)
            
            if current_error < target_error:
                print(f"✓ Converged at iteration {iteration} (error: {current_error:.4f})")
                break
            
            # Find best candidate
            best_unit, new_error = self.find_best_unit(data, targets)
            
            if best_unit is None or new_error >= current_error:
                print(f"✗ No improvement at iteration {iteration}")
                break
            
            # Add best unit and freeze it
            self.frozen_hidden.append(best_unit)
            print(f"Iteration {iteration}: Added unit (error: {new_error:.4f})")
            
            # Retrain output weights
            total_units = len(self.frozen_hidden) + len(self.trainable_hidden)
            self.output_weights = np.random.randn(total_units, self.n_outputs)
            # Simple pseudo-inverse solution
            A = self._compute_output(data)
            try:
                self.output_weights = np.linalg.pinv(A) @ targets
            except:
                pass
    
    def predict(self, x):
        """Predict output for new input"""
        return self._compute_output(x)[0]

# Run the example
if __name__ == "__main__":
    network = CascadeCorrelationNetwork(n_inputs=2, n_outputs=1, hidden_capacity=5)
    network.grow(X_train, y_train, max_iterations=10, target_error=0.1)
    
    print("\nTesting:")
    for x, y in X:
        pred = network.predict(x)
        print(f"  [{x[0]}, {x[1]}] → {pred:.3f} (expected: {y})")

Key insights from this implementation:

  • Units start by connecting to pairs of inputs
  • Each unit is trained in isolation before being frozen
  • Output weights are retrained after each unit addition
  • The loop stops when error is acceptable or no improvement is found
