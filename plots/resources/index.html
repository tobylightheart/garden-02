Resources

Curated Learning Resources

A collection of books, courses, tools, and documentation to support your learning journey. These are resources I've found valuable in my exploration. Each resource is included because it provided genuine insight, practical value, or helped clarify complex concepts.

Learning Philosophy

🎯 How I Choose Resources

  • Practical Application: Does it help me build or understand something concrete?
  • Clarity: Is the explanation clear and accessible?
  • Depth: Does it go beyond surface-level explanations?
  • Timelessness: Will it remain relevant years from now?

This curation focuses on resources that have stood the test of time and provided genuine value in my learning journey.

Books

Books that have shaped my understanding of programming, learning, and thinking.

  • 📖 "Clean Code" by Robert C. Martin

    What I learned: Principles of writing maintainable, readable code that other developers can understand and build upon. Key takeaways include meaningful names, small functions, and the importance of code as communication. Essential for any developer who wants to write code that lasts.

    Programming Intermediate
  • 📖 "The Pragmatic Programmer" by Andrew Hunt & David Thomas

    What I learned: Timeless advice for software developers that applies regardless of language or framework. Covers topics like automation, design patterns, continuous integration, and the importance of understanding requirements. A must-read that I return to regularly as I face new challenges.

    Software Development All Levels
  • 📖 "Design Patterns" by Gang of Four (Erich Gamma, Richard Helm, Ralph Johnson, John Vlissides)

    What I learned: Classic patterns for object-oriented design that solve recurring problems. The Singleton, Factory, Observer, and Strategy patterns have become staples in my code. Though dated in some examples, the core concepts remain timeless and applicable across languages.

    Software Architecture Advanced
  • 📖 "Thinking, Fast and Slow" by Daniel Kahneman

    What I learned: How we think and make decisions, revealing the two systems of thought: fast, intuitive System 1 and slow, deliberate System 2. Understanding these systems helps me recognize cognitive biases in my own thinking and decision-making.

    Psychology Accessible
  • 📖 "You Don't Know JS" by Kyle Simpson

    What I learned: Deep dive into JavaScript's quirks, closures, prototypal inheritance, and the event loop. This series transformed my understanding of how JavaScript actually works under the hood, rather than just how the syntax looks.

    JavaScript Advanced

Courses

Structured courses that provided systematic learning paths and deep dives into specific topics.

  • 🎓 Computer Science: Basic Algorithms and Data Structures (Coursera / Princeton)

    Why it's valuable: Fundamental algorithms and data structures with Java implementations. Covers sorting, searching, graphs, trees, and dynamic programming with rigorous mathematical proofs. The algorithmic thinking skills here are transferable to any programming language.

    Computer Science 6 weeks
  • 🎓 Machine Learning Specialization (Coursera / Andrew Ng)

    Why it's valuable: ML fundamentals from Andrew Ng, the creator of the course. Covers supervised and unsupervised learning, neural networks, and practical implementation. The mathematical foundation is solid without being overwhelming.

    Machine Learning 6 weeks
  • 🎓 Full Stack Open (University of Helsinki)

    Why it's valuable: Modern JavaScript and React development, covering the full stack from frontend to backend. Includes Node.js, Express, GraphQL, and testing practices. Updated regularly to stay current with modern web development practices.

    Web Development 4 weeks
  • 🎓 Deep Learning Specialization (Coursera / Andrew Ng)

    Why it's valuable: Advanced deep learning covering CNNs, RNNs, transformers, and practical applications. Builds on the Machine Learning Specialization and dives deeper into neural network architectures.

    Deep Learning 4 weeks

Learning Strategies

🔍 Learning by Documentation

Some of the most valuable learning comes from reading documentation carefully rather than executing code immediately. Documentation teaches you the concepts and mental models behind the tools. When you understand "why" something works, you're better equipped to troubleshoot when it doesn't.

My approach: Read the documentation → Try the examples → Experiment with modifications → Understand the underlying principles.

🔍 Learning by Doing

Build projects, break things, fix them. Hands-on experience cements understanding better than passive consumption. The Digital Garden itself is a product of this approach - I learn by documenting what I learn, which forces me to clarify my understanding.

My approach: Start small → Build iteratively → Break things intentionally → Learn from failures → Document the journey.

🔍 Learning by Teaching

Explaining concepts to others (or even to yourself) reveals gaps in understanding. Writing documentation, creating tutorials, or simply writing clear notes forces you to organize your knowledge coherently. This is the Feynman technique in action - if you can't explain it simply, you don't understand it well enough.

My approach: Write documentation → Create examples → Share with others → Get feedback → Refine understanding.
