Git Workflow

Version Control Best Practices

Git is more than a tool - it's a workflow. Understanding how to use Git effectively helps you collaborate better, track changes, and maintain clean project histories.

Core Concepts

🔄 Git Model

Git has three main states:

  • Working Directory: Your actual files
  • Staging Area: Files ready to commit
  • Repository: Committed history

Basic Commands

📥 Getting Started

  • git init: Start a new repository
  • git clone <url>: Copy from remote
  • git status: See current state
  • git add <file>: Stage changes
  • git commit -m "msg": Create a commit
  • git push: Upload to remote
  • git pull: Download from remote

🔍 Viewing History

  • git log: Show commit history
  • git log --oneline: Compact view
  • git diff: Show unstaged changes
  • git diff --staged: Show staged changes
  • git show <commit>: View commit details

Branching Workflow

🌿 Branching Model

A good workflow uses branches for organization:

  • main: Stable, production-ready code
  • develop: Integration branch
  • feature/<name>: New features
  • bugfix/<name>: Bug fixes

🔄 Branch Operations

  • git branch <name>: Create branch
  • git checkout -b <name>: Create and switch
  • git merge <branch>: Merge branch
  • git rebase <branch>: Rebase commits
  • git cherry-pick <commit>: Apply commit

Pull Request Workflow

📤 Pull Request Process

  1. Create branch: git checkout -b feature/xyz
  2. Make changes: Code, test, commit
  3. Push to remote: git push origin feature/xyz
  4. Create PR: On GitHub/GitLab
  5. Review: Get feedback
  6. Update: Fix issues, rebase if needed
  7. Merge: Once approved

🔒 Good PR Practices

  • Small commits: One change per commit
  • Clear messages: Describe WHAT and WHY
  • Tests included: Prove it works
  • Code review: Let others check
  • Fix feedback: Address all comments

Commit Message Best Practices

📝 Conventional Commits

Format: <type>(optional scope): <description>

  • Types:
    • feat: New feature
    • fix: Bug fix
    • docs: Documentation
    • style: Formatting
    • refactor: Code restructuring
    • test: Tests
    • chore: Maintenance
  • Scope: What part of the code (optional)
  • Description: Clear, imperative tense

📋 Example Commit Messages

  • feat(data): add data structures page
  • fix(python): correct indentation in example
  • docs(readme): update installation instructions
  • refactor(css): simplify flexbox layout
  • test(sorting): add unit tests for merge sort

Advanced Techniques

🔙 Undoing Changes

  • git restore <file>: Undo unstaged changes
  • git reset HEAD <file>: Unstage file
  • git revert <commit>: Undo a commit
  • git revert -n <commit>: Undo without new commit

🎯 Stashing Changes

  • git stash: Save uncommitted changes
  • git stash list: View stashes
  • git stash pop: Apply and remove stash
  • git stash push "msg": Save with message

🤔 Git Best Practices

  • Commit often: Small, focused commits
  • Write good messages: Explain why, not just what
  • Branch for changes: Never edit main directly
  • Keep branches small: Easy to review
  • Rebase before merging: Clean history
  • Use meaningful names: Descriptive branches
  • Protect main: Require PRs for main
