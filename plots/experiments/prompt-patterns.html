Prompt Patterns

Mastering Prompt Engineering

Prompts are the interface between humans and AI. Good prompts elicit helpful, accurate, and creative responses. Understanding prompt patterns helps you communicate effectively with large language models.

Core Prompt Structure

Every good prompt has three parts:

  • Role: Who should the AI be? (expert, teacher, critic, etc.)
  • Task: What should it do? (clear, specific instructions)
  • Context: What background info helps? (examples, constraints, format requirements)

Example: "As a senior software architect (role), review this code for security vulnerabilities (task). The code handles user authentication (context). Use Python and provide specific line numbers (format)."

Essential Patterns

🎯 Zero-Shot Prompting

Direct request without examples. Best for common tasks, straightforward questions, or when you trust the model's general knowledge.

Simple example:

"Explain quantum entanglement in simple terms."

Advanced example:

"You are a history professor. Explain the causes of World War I in three paragraphs, focusing on: 1) the assassination of Archduke Franz Ferdinand, 2) alliance systems, and 3) economic competition."

📌 When to Use
  • Common, straightforward tasks
  • When you don't have specific examples to share
  • Simple questions where the answer is well-defined
  • Initial exploration of a topic

📝 Few-Shot Prompting

Provide examples before the task. This is one of the most powerful techniques for improving accuracy, especially for complex tasks, style transfer, or format specification.

Example - Translation:

Translate these phrases to Spanish:\n\n "Hello" → "Hola"\n "Goodbye" → "Adiós"\n "Thank you" → "Gracias"\n "How are you?" → ?

Example - Code Generation:

Write Python functions to calculate area:\n\n def area_circle(r):\n return π * r²\n \n def area_rectangle(w, h):\n return w * h\n \n def area_triangle(b, h):\n return 0.5 * b * h

Best for: Style transfer, format specification, pattern recognition, code generation, and tasks where the pattern is clear from examples.

📌 When to Use
  • You have clear examples of the desired output
  • The task requires following a specific format or style
  • Pattern-based tasks (translation, classification, etc.)
  • When zero-shot prompts give inconsistent results

📋 Chain-of-Thought

Ask the model to think step by step. This dramatically improves performance on reasoning tasks, math problems, and logic puzzles by forcing the model to show its work rather than jumping to conclusions.

Example - Math Problem:

I have 3 apples. I eat 1 and buy 4 more. Then I give away 2. How many do I have? Think step by step and show your work.

Example - Logic Puzzle:

Three people make these statements:\n - Alice: "Bob is lying"\n - Bob: "Charlie is lying"\n - Charlie: "Both Alice and Bob are lying"\n \n Only one person is telling the truth. Who is it? Think through each possibility step by step.

📌 When to Use
  • Mathematical problems and calculations
  • Logic puzzles and reasoning tasks
  • Complex decisions requiring multiple steps
  • When accuracy is critical and you can verify the reasoning

🎭 Role-Playing

Assign a persona to improve quality. Different roles trigger different knowledge bases, tones, and perspectives, helping you get more relevant and useful responses.

Example - Code Review:

Act as a senior software architect with 15 years of experience in Python web development. Review this code and suggest improvements focused on: security, performance, and maintainability. Be specific about potential issues and provide concrete alternatives.

Example - Creative Writing:

You are a bestselling science fiction author known for hard science fiction with deep philosophical undertones. Write a 500-word story about a spaceship discovering a signal from another civilization. Include themes of first contact and the uncertainty of alien intent.

📌 When to Use
  • Specialized tasks requiring domain expertise
  • When you need a specific tone or perspective
  • Creative writing and brainstorming
  • When the AI's default response is too generic

🔄 Iterative Refinement

Start broad, then narrow down. Don't expect perfection on the first try - treat prompts as conversations where you refine based on initial results.

Example - Multi-round refinement:

First prompt: "Write a story about time travel."\n \n AI response: Generic story gets written.\n \n Second prompt: "Rewrite with a physicist protagonist. The time travel involves quantum mechanics. Write in first person from their perspective."\n \n AI response: Better, more specific story.\n \n Third prompt: "The protagonist discovers they're the same person they met at the beginning. End on a cliffhanger where they realize their entire life was an experiment."\n \n AI response: Deep, thought-provoking story with a twist ending.

📌 When to Use
  • Complex creative writing tasks
  • When you need to explore ideas before committing
  • Tasks requiring multiple levels of detail
  • When the first attempt misses key requirements

📊 Output Formatting

Specify the exact format you want. LLMs are excellent at following format instructions when they're clear and explicit.

Example - Table:

List the pros and cons of Python vs JavaScript in a markdown table with columns: Feature, Python, JavaScript. Include these features: syntax, performance, ecosystem, use cases, learning curve.

Example - JSON:

Extract all product names and prices from this text and return them as valid JSON:\n \n [Your text here]\n \n Format: {"products": [{"name": "...", "price": ...}]}

Best for: Structured output, data extraction, comparisons, and when you need to integrate AI output into other systems.

📌 When to Use
  • When you need machine-readable output
  • Data extraction from unstructured text
  • Creating comparisons or summaries
  • Integrating AI into automated workflows

Advanced Techniques

🔗 Context Anchoring

Ground responses in specific knowledge. Provide code snippets, documents, or background information that should inform the response.

Example:

Based on the following Python code:\n \n [code block]\n \n Explain what this function does in exactly 3 bullet points. Include:\n 1. The overall purpose\n 2. Key parameters and their effects\n 3. Return value and type

🎪 Constraint Setting

Define what NOT to do. Constraints often improve quality by eliminating common mistakes or unwanted patterns.

Example:

Explain blockchain technology to a 12-year-old. Don't use jargon. Keep it under 200 words. Use an analogy involving shipping containers and tracking. Avoid mentioning cryptography.

Example - Code Generation:

Write a Python function to validate an email address. Don't use regular expressions. Use only basic string operations. Handle edge cases like empty strings and missing @ symbol.

📌 When to Use
  • When the default response includes unwanted content
  • When you need to avoid common pitfalls
  • For children or non-expert audiences
  • When format or length is critical

🔄 Self-Correction

Ask the model to critique its own output. This leverages the model's ability to find errors and improve quality through reflection.

Example - Writing:

Write a blog post about neural networks for beginners. Then critique it for:\n 1. Clarity - Are complex concepts explained simply?\n 2. Accuracy - Are there any technical errors?\n 3. Engagement - Is it interesting or dry?\n \n Rewrite with improvements, addressing all three criteria.

Example - Code:

Create a Python function that sorts a list using bubble sort.\n \n 1. Write the function\n 2. Generate 10 test cases with edge cases (empty list, single element, already sorted, reverse sorted, duplicates)\n 3. Run the tests and report any failures\n 4. Fix bugs based on test results\n 5. Write documentation and add type hints

📌 When to Use
  • When accuracy is critical
  • For code that will be used in production
  • When you need multiple iterations of improvement
  • Writing for specific audiences

🤔 Prompt Best Practices

  • Be specific: "Write a poem" → "Write a haiku about rain in Tokyo, using classical Japanese imagery"
  • Provide examples: Show, don't just tell - few-shot is often better than zero-shot
  • Set constraints: Word limits, format, tone, what to avoid
  • Iterate: Treat prompts as conversations, not one-off commands
  • Test variations: Try different phrasings if the first attempt isn't ideal
  • Separate context from task: Clear boundaries help the model understand what's background and what's the actual request
  • Be explicit about format: If you need JSON, tables, code, or specific structures, say so clearly

Common Pitfalls to Avoid

⚠️ Vague Instructions

"Write something about AI" → Too vague, likely to get generic responses.\n "Write a comprehensive guide on prompt engineering for intermediate users, covering zero-shot, few-shot, and chain-of-thought techniques with practical examples" → Much better!

⚠️ Contradictory Constraints

"Write a very long essay that's under 500 words" → Confusing! Pick one.\n "Be formal and casual at the same time" → Choose a tone.

⚠️ Assuming the AI Knows Your Context

Don't say "Write code like I do" - the AI doesn't know your style.\n Instead: "Write Python code following PEP 8 conventions with detailed docstrings."
