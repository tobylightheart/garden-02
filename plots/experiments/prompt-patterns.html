Prompt Patterns

Mastering Prompt Engineering

Prompts are the interface between humans and AI. Good prompts elicit helpful, accurate, and creative responses. Understanding prompt patterns helps you communicate effectively with large language models.

Core Prompt Structure

Every good prompt has three parts:

  • Role: Who should the AI be? (expert, teacher, etc.)
  • Task: What should it do? (clear, specific instructions)
  • Context: What background info helps? (examples, constraints)

Essential Patterns

🎯 Zero-Shot Prompting

Direct request without examples.

Example:

"Explain quantum entanglement in simple terms."

Best for: Common tasks, straightforward questions

📝 Few-Shot Prompting

Provide examples before the task.

Example:

Translate these:\n "Hello" → "Hola"\n "Goodbye" → "Adiós"\n "Thank you" → ?

Best for: Style transfer, format specification, pattern recognition

📋 Chain-of-Thought

Ask the model to think step by step.

Example:

"I have 3 apples. I eat 1 and buy 4 more. Then I give away 2. How many do I have? Think step by step."

Best for: Reasoning tasks, math, logic problems

🎭 Role-Playing

Assign a persona to improve quality.

Example:

"Act as a senior software architect. Review this code and suggest improvements."

Best for: Specialized tasks, getting expert perspectives

🔄 Iterative Refinement

Start broad, then narrow down.

Example:

First: "Write a story about time travel." \n\n Then: "The story has a physicist protagonist. The time travel involves quantum mechanics. Write in first person." \n\n Then: "The protagonist discovers they're the same person they met at the beginning. End on a cliffhanger."

Best for: Creative writing, complex tasks

📊 Output Formatting

Specify the exact format you want.

Example:

"List the pros and cons of Python vs JavaScript in a markdown table with columns: Feature, Python, JavaScript."

Best for: Structured output, data extraction, comparisons

Advanced Techniques

🔗 Context Anchoring

Ground responses in specific knowledge.

Example:

"Based on the following Python code:\n [code block]\n Explain what this function does in exactly 3 bullet points."

🎪 Constraint Setting

Define what NOT to do.

Example:

"Explain blockchain technology. Don't use jargon. Keep it under 200 words. Use an analogy involving shipping."

🔄 Self-Correction

Ask the model to critique its own output.

Example:

"Write a blog post about neural networks. Then critique it for clarity, accuracy, and engagement. Rewrite with improvements."

🤔 Prompt Best Practices

  • Be specific: "Write a poem" → "Write a haiku about rain in Tokyo"
  • Provide examples: Show, don't just tell
  • Set constraints: Word limits, format, tone
  • Iterate: Refine based on output
  • Test variations: Try different phrasings
  • Separate context from task: Clear boundaries
