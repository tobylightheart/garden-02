LLM Testing

Evaluating Large Language Models

Testing and evaluating large language models (LLMs) is both an art and a science. How do we know an LLM is working well? What benchmarks matter? And how do we test capabilities that don't fit into standard tests?

Benchmark Categories

📝 Standardized Benchmarks

Widely-used evaluation suites:

  • MMLU: Massive Multitask Language Understanding (57 tasks)
  • GSM8K: Grade school math word problems
  • HumanEval: Code generation tasks
  • MATH: High school and competition math

Limitations: Can game benchmarks with memorization

🎯 Capability-Specific Tests

Tests designed for specific abilities:

  • Reasoning: Chain of thought, multi-step problems
  • Factual: Knowledge accuracy and recency
  • Code: Syntax, debugging, algorithm generation
  • Language: Translation, grammar, style
  • Reasoning about text: Reading comprehension, inference

🔬 Human Evaluation

Automated tests miss nuances. Human evaluators assess:

  • Helpfulness: Does it actually help the user?
  • Harmlessness: Is it safe and ethical?
  • Honesty: Does it admit uncertainty?
  • Coherence: Is reasoning sound?

Testing Strategies

🎲 Adversarial Testing

Push the model to its limits:

  • Edge cases: Ambiguous, contradictory, or confusing prompts
  • Jailbreak attempts: Testing safety boundaries
  • Reasoning traps: Problems requiring multi-hop reasoning
  • Hallucination tests: Facts that don't exist

🧩 Chain-of-Thought Testing

Evaluate reasoning step-by-step:

  • COT: Ask model to show its work
  • Self-correction: Can it find and fix its errors?
  • Multi-hop: Questions requiring multiple inferences

📊 Regression Testing

Track improvements and detect regressions:

  • Test suite: Fixed set of questions
  • Score tracking: Monitor performance over time
  • Comparative analysis: Compare model versions

Common Test Types

🔢 Math & Reasoning

Test numerical and logical abilities:

  • Word problems requiring multi-step reasoning
  • Mathematical proofs
  • Logic puzzles
  • Scientific reasoning

💻 Code Generation

Evaluate programming capabilities:

  • Function generation from specifications
  • Debugging existing code
  • Algorithm implementation
  • Code explanation and refactoring

📚 Knowledge & Facts

Test factual accuracy:

  • General knowledge questions
  • Technical documentation understanding
  • Domain-specific expertise
  • Fact-checking capabilities

🎨 Creative Tasks

Evaluate creativity and style:

  • Writing stories and poems
  • Style transfer and imitation
  • Idea generation
  • Creative problem-solving

🤔 Evaluation Challenges

Testing LLMs isn't straightforward:

  • Benchmark gaming: Models memorize answers
  • Context sensitivity: Performance varies by prompt
  • Subjectivity: Hard to quantify "good" answers
  • Emergent abilities: New skills appear unexpectedly
  • Alignment gaps: Scores don't predict real-world usefulness

The best evaluation combines automated benchmarks, human judgment, and real-world usage testing.

Setting Up an LLM Testing Pipeline

🔧 Building a Test Framework

Here's a practical Python framework for testing LLMs with customizable benchmarks:

Test Suite Manager
import json
import random
from dataclasses import dataclass
from typing import List, Dict, Callable, Optional
from functools import wraps

@dataclass
class TestQuestion:
    """A single test question"""
    id: str
    prompt: str
    expected_keywords: List[str]  # Keywords to check in response
    max_tokens: int = 1000
    category: str = "general"
    
    def to_dict(self):
        return {
            'id': self.id,
            'prompt': self.prompt,
            'expected_keywords': self.expected_keywords,
            'max_tokens': self.max_tokens,
            'category': self.category
        }

@dataclass
class TestResults:
    """Results for a single test run"""
    question_id: str
    model_name: str
    response: str
    score: float  # 0.0 to 1.0
    matched_keywords: List[str]
    total_keywords: int
    tokens_used: int
    errors: List[str]

class LLMTestSuite:
    """Manages a collection of LLM test questions and evaluates models"""
    
    def __init__(self, questions: List[TestQuestion]):
        self.questions = questions
        self.results_history = []
    
    def add_question(self, question: TestQuestion):
        """Add a new test question to the suite"""
        self.questions.append(question)
    
    def randomize(self):
        """Shuffle questions for varied testing"""
        random.shuffle(self.questions)
    
    def evaluate_response(self, response: str, expected_keywords: List[str]) -> float:
        """Calculate how well response matches expected keywords"""
        response_lower = response.lower()
        matched = sum(1 for kw in expected_keywords if kw.lower() in response_lower)
        return matched / len(expected_keywords)
    
    def run_test(self, question: TestQuestion, model_response: str) -> TestResults:
        """Run a single test and return results"""
        expected_keywords = question.expected_keywords
        matched = []
        
        for kw in expected_keywords:
            if kw.lower() in model_response.lower():
                matched.append(kw)
        
        score = self.evaluate_response(model_response, expected_keywords)
        
        return TestResults(
            question_id=question.id,
            model_name="test_model",
            response=model_response[:200],  # Truncate for display
            score=score,
            matched_keywords=matched,
            total_keywords=len(expected_keywords),
            tokens_used=question.max_tokens,
            errors=[]
        )
    
    def run_suite(self, model_response_fn, num_retries=2) -> List[TestResults]:
        """Run all tests with a model (function that returns response)"""
        results = []
        
        for question in self.questions:
            for attempt in range(num_retries):
                try:
                    response = model_response_fn(question)
                    break
                except Exception as e:
                    if attempt == num_retries - 1:
                        results.append(TestResults(
                            question_id=question.id,
                            model_name="test_model",
                            response=f"Error: {str(e)}",
                            score=0.0,
                            matched_keywords=[],
                            total_keywords=len(question.expected_keywords),
                            tokens_used=0,
                            errors=[f"Failed after {num_retries} retries: {e}"]
                        ))
                        continue
            
            results.append(self.run_test(question, response))
        
        return results

📈 Benchmark Aggregation and Visualization

Track performance across multiple benchmarks over time:

Performance Tracker
import json
from dataclasses import dataclass, asdict
from typing import List, Dict
from datetime import datetime

@dataclass
class BenchmarkResult:
    """Results for one benchmark run"""
    benchmark_name: str
    model_version: str
    date: str
    score: float
    samples: int
    time_taken: float
    details: Dict[str, float]  # Per-task scores
    
    def to_dict(self):
        return asdict(self)

class BenchmarkTracker:
    """Track and aggregate benchmark results"""
    
    def __init__(self):
        self.results: List[BenchmarkResult] = []
    
    def add_result(self, result: BenchmarkResult):
        self.results.append(result)
    
    def get_average_score(self, benchmark_name: str) -> float:
        """Get average score for a specific benchmark"""
        benchmark_results = [r for r in self.results if r.benchmark_name == benchmark_name]
        if not benchmark_results:
            return 0.0
        return sum(r.score for r in benchmark_results) / len(benchmark_results)
    
    def get_model_comparison(self, model_versions: List[str]) -> Dict[str, float]:
        """Compare average scores across model versions"""
        comparison = {}
        for version in model_versions:
            scores = []
            for result in self.results:
                if result.model_version == version:
                    scores.append(result.score)
            comparison[version] = sum(scores) / len(scores) if scores else 0.0
        return comparison
    
    def export_to_json(self, filename: str):
        """Export results to JSON file"""
        with open(filename, 'w') as f:
            json.dump([r.to_dict() for r in self.results], f, indent=2)
    
    def print_summary(self):
        """Print a text summary of results"""
        print("=" * 60)
        print("BENCHMARK SUMMARY")
        print("=" * 60)
        
        benchmarks = set(r.benchmark_name for r in self.results)
        for benchmark in sorted(benchmarks):
            benchmark_results = [r for r in self.results if r.benchmark_name == benchmark]
            avg_score = sum(r.score for r in benchmark_results) / len(benchmark_results)
            print(f"\n{benchmark}:")
            print(f"  Average score: {avg_score:.2%}")
            print(f"  Runs: {len(benchmark_results)}")
            for r in benchmark_results[:3]:  # Show first 3
                print(f"    - {r.model_version}: {r.score:.2%}")
        
        print("\n" + "=" * 60)

# Example usage
