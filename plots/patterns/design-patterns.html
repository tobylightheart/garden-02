Design Patterns

Software Design Patterns

Design patterns are recurring solutions to common problems in software design. They capture best practices that have been tested and refined over time.

The Three Categories

Design patterns are typically divided into three groups:

📦 Creational Patterns

Focus on object creation mechanisms. They promote loose coupling by decoupling a system from how its objects are created.

  • Singleton: Ensures a class has only one instance
  • Factory: Creates objects without specifying exact class
  • Builder: Constructs complex objects step by step
  • Prototype: Creates new objects by copying an existing prototype

🔗 Structural Patterns

Focus on how classes and objects are composed to form larger structures.

  • Adapter: Makes incompatible interfaces work together
  • Decorator: Adds responsibilities to objects dynamically
  • Facade: Provides simplified interface to complex subsystem
  • Proxy: Controls access to another object

🔄 Behavioral Patterns

Focus on communication between objects and how responsibilities are assigned.

  • Observer: Defines dependency between objects for state changes
  • Strategy: Defines family of algorithms, making them interchangeable
  • Iterator: Provides sequential access to elements without exposing structure
  • Memento: Captures and restores internal state

Python Example: Singleton Pattern

class Singleton:
    _instance = None
    _initialized = False
    
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance
    
    def __init__(self, value=None):
        if not self._initialized:
            self.value = value
            self._initialized = True
    
    def get_value(self):
        return self.value

This ensures only one instance exists, while allowing initialization with different values.

When to Use Patterns

Patterns are powerful, but they're not silver bullets:

  • Use patterns when you're solving a recurring problem
  • Don't overuse - simple solutions are often best
  • Understand the trade-offs each pattern introduces
  • Document why you chose a particular pattern

💡 Pattern Recognition

Learning design patterns helps you recognize patterns in problems and apply proven solutions. Like learning to read music helps a musician recognize melodies, pattern recognition accelerates problem-solving.
